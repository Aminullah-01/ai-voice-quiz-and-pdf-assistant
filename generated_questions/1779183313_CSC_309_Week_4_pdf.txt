This presentation uses a free template provided by FPPT.com www.free -power -point -templates.comComponents of Rule based Systems •A rule -based system consists of a number of components: •A database of rules (also called a knowledge base ) •the knowledge base consists of a set of rules that represent the knowledge that the system has •A database of facts (also called as working Memory) •The database of facts represents inputs to the system that are used to derive conclusions, or to cause actions •An interpreter , or inference engine •The interpreter, or inference engine, is the part of the system that controls the process of deriving conclusions. It uses the rules and facts, and combines them together to draw conclusions. This presentation uses a free template provided by FPPT.com www.free -power -point -templates.comBasic inference procedure in rule -based system •The operation of a rule -based system follows a cyclic process: •Matching: The inference engine compares the conditions of each rule with the contents of the working memory to identify rules whose conditions are met. •Selection: Among the rules whose conditions are satisfied, the system selects the rules to be executed. This can involve prioritizing rules, resolving conflicts, and determining the appropriate sequence of rule execution. •Execution: The selected rules' associated actions are executed, leading to changes in the working memory or generating responses or outputs. •Updating: After rule execution, the working memory may be updated based on the changes brought about by the executed actions. •The cycle then repeats as the system continues to process new information and make decisions . The system has a set of rules that link specific symptoms to potential diseases. Here's how forward chaining works: •Initial Facts: The system starts with the patient's symptoms as input data (e.g., fever, cough, headache). •Rule Application: The rules are applied to the given symptoms. For instance: •If fever AND cough, then possibility of flu. •If headache AND runny nose, then possibility of sinusitis. •New Conclusions: Based on the given symptoms and rule application, the system identifies potential diseases (e.g., flu, sinusitis). •Iteration: If the system has more specific rules related to each potential disease, it can iteratively apply those rules to gather more evidence and refine the diagnosis. It may take a lot of time to eliminate and synchronize available data. •Unlike backward chaining, the explanation of facts or observations for this type of chaining is not very clear. The former uses a goal -driven method that arrives at conclusions efficiently. This presentation uses a free template provided by FPPT.com www.free -power -point -templates.comBackward Chaining •Backward chaining, also known as goal -driven reasoning or top -down reasoning, begins with a specific goal or conclusion and works backward through the rules to determine the necessary conditions or facts that must be satisfied to reach that goal. • i.e. you starts from a conclusion/goal and move backward through the rules to identify the facts that led to that conclusion in the database of facts • best for diagnosis problems. This presentation uses a free template provided by FPPT.com www.free -power -point -templates.comBackward Chaining Example •Example: Automated Diagnosis •Consider an automated diagnosis system for computer hardware issues. The goal is to identify the root cause of a malfunction. •Rules: •IF no power THEN check power supply. •IF overheating THEN clean the fan and heat sink. •IF slow performance AND overheating THEN apply thermal paste. •Goal: slow performance •Rule 3 is a possible solution for the goal. •Condition of rule 3 is not met (overheating), so we apply backward chaining to satisfy the condition. •Rule 2 addresses overheating. •Apply rule 2: overheating → clean fan and heat sink. •The backward chaining process helps identify the necessary steps to address the goal. This presentation uses a free template provided by FPPT.com www.free -power -point -templates.comBackward Chaining: Pros and Cons •Advantages •The result is already known, which makes it easy to deduce inferences. •It’s a quicker method of reasoning than forward chaining because the endpoint is available. •In this type of chaining, correct solutions can be derived effectively if pre - determined rules are met by the inference engine. •Disadvantages •The process of reasoning can only start if the endpoint is known. •It doesn’t deduce multiple solutions or answers. •It only derives data that is needed, which makes it less flexible than forward chaining.
